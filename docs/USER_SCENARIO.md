# 마이링크 (MyLink) 사용자 시나리오

본 문서는 마이링크 서비스의 주요 사용자(페이지 방문객 및 페이지 소유자)가 목적을 달성하기 위해 수행하는 행동 흐름을 정의합니다.

모든 시나리오는 요청하신 `"사용자는 __하기 위해 __한다"`의 포맷에 맞추어 작성되었습니다.

---

## 1. 방문자 (Visitor) 시나리오: 마이링크 페이지 방문 과정

### 1-1. 페이지 접속 및 정보 확인
- 방문자는 **페이지 소유자의 다양한 활동 정보를 한눈에 알아보기 위해** 전달받은 고유 URL(`mylink.com/닉네임`)을 클릭하여 웹페이지에 접속한다.
- 방문자는 **해당 페이지가 누구의 것인지 인지하기 위해** 화면 최상단에 렌더링된 소유자의 유저네임(실제 이름), 구글 프로필 이미지, 한 줄 소개(Bio)를 시각적으로 확인한다.

### 1-2. 개별 링크 클릭 및 이동
- 방문자는 **리스트 중에서 자신이 원하는 특정 서비스나 콘텐츠로 이동하기 위해** 세로로 나열된 링크 버튼의 '제목(Title)'과 좌측에 동적으로 렌더링된 '파비콘(Favicon)'을 살펴본다.
- 방문자는 **해당 도메인이나 목적지로 실제로 넘어가기 위해** 마음에 드는 링크 버튼을 클릭하고, 새 탭(또는 새 창)으로 열리는 외부 웹사이트를 탐험한다.

### 1-3. 프로필 복사 및 메인 서비스 전환
- 방문자는 **이 유용하고 예쁜 링크 모음을 다른 친구나 지인에게 공유하기 위해** 페이지에 위치한 `페이지 링크 복사` 버튼을 클릭한다.
- 방문자는 **자신도 이렇게 깔끔한 개인 링크 트리를 만들어보기 위해** 페이지 최하단에 작게 노출된 `Powered by MyLink` 워터마크(버튼)를 클릭하여 마이링크 메인 랜딩 페이지(회원가입 창)로 이동한다.

### 1-4. 잘못된 주소 접속 (예외 시나리오)
- 방문자는 **정확한 서비스의 정체를 인지하고 이탈을 방지하기 위해** 존재하지 않거나 오타가 발생한 URL에 접속했을 경우, 단순 브라우저 에러가 아닌 "존재하지 않는 페이지입니다. 나만의 마이링크를 만들어보세요!" 라는 문구가 포함된 **자체 404 에러 단독 페이지**를 확인한다.

---

## 2. 소유자 (Owner) 시나리오: 로그인/아웃 및 메뉴 관리 과정

### 2-1. 회원가입 및 소셜 로그인 연동
- 소유자(가입 예정자)는 **나만의 깔끔한 링크 페이지를 개설하거나 기존 대시보드에 접근하기 위해** 마이링크 메인 화면에서 `구글 로그인으로 시작하기` 버튼을 클릭한다.
- 소유자는 **별도의 복잡한 회원가입 폼(이메일 작성, 비밀번호 설정)을 우회하기 위해** 팝업창에서 자신이 사용하는 구글 계정을 선택해 즉각적인 연동 인증(가입 및 로그인)을 완료한다.

### 2-2. 관리자 대시보드 진입 및 내 주소 복사
- 소유자는 **자신의 링크 리스트를 최신 상태로 관리하기 위해** 로그인을 정상적으로 마친 후 대시보드 화면으로 자동 진입한다.
- 소유자는 **자신의 현재 페이지 URL을 복사해 인스타그램 프로필 등에 빠르게 붙여넣기 위해** 대시보드 상단 영역에 노출된 `내 주소 복사` 버튼을 클릭한다.

### 2-3. 빈 상태(Empty State)에서의 초기화면 확인
- 소유자는 **링크를 처음에 어떻게 만들어야 하는지 직관적으로 인지하기 위해** 가입 직후 초기 화면에서 아무런 가짜 데모 항목 없이, "아직 등록된 링크가 없습니다"라는 빈 안내 문구와 메인 `새 링크 추가하기` 버튼만을 확인한다.

### 2-4. 새로운 링크 항목 추가
- 소유자는 **방문자들에게 뽐내고 싶은 새로운 웹사이트의 경로를 알리기 위해** 대시보드의 링크 리스트 하단에 위치한 `새 링크 추가하기` 버튼을 클릭한다.
- 소유자는 **방문자가 해당 링크를 명확히 이해할 수 있도록 정의하기 위해** 새롭게 생성된 빈 목록 칸에 마우스 커서를 올리고 '제목(Title)'과 '도메인 주소(URL)'를 직접 타이핑한다.
- 소유자는 **작성한 내용을 시스템에 최종 반영하기 위해** 키보드의 `Enter` 키를 누르거나 입력 영역 바깥(배경 빈 공간)을 클릭하여 인라인 입력을 종료 및 자동 저장시킨다.

### 2-5. 기존 링크 항목 수정 (인라인 에디팅)
- 소유자는 **과거에 등록해둔 링크의 제목이나 목적지 주소가 변경되었음을 알리기 위해** 링크 리스트 중 고치고 싶은 항목의 텍스트가 표시된 영역을 마우스로 클릭한다.
- 소유자는 **오타를 바로잡거나 새로운 주소로 갱신하기 위해** 인라인(Inline) 입력창으로 폼이 즉시 변환된 상태에서 텍스트를 수정하고 다시 입력을 완료(Enter/블러 처리)한다.

### 2-6. 링크 항목 삭제
- 소유자는 **더 이상 유효하지 않거나 노출하고 싶지 않은 링크를 제거하기 위해** 해당 링크 항목 우측 끝 영역에 표시된 `삭제(휴지통)` 아이콘 버튼을 찾아 클릭한다.
- 소유자는 **혹시 모를 마우스 조작 실수로 중요 링크가 지워지는 것을 방지하기 위해** 화면에 나타난 확인 알림창(Confirm Alert)에서 '네, 삭제합니다' 버튼을 클릭하여 항목을 영구적으로 삭제한다.

### 2-7. 안전한 세션 종료 (로그아웃)
- 소유자는 **공용 PC 등에서 자신의 계정 통제 권한을 안전하게 보호하기 위해** 대시보드 화면 상단 코너에 있는 `로그아웃` 버튼을 클릭한다.
- 소유자는 **세션이 정상적으로 완전히 종료되었음을 인지하기 위해** 로그아웃 시도 직후, 서비스의 메인 랜딩 페이지(버튼이 있는 초기 화면)로 튕겨져 나오는 화면 전환을 눈으로 확인한다.
