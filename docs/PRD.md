# 마이링크 (MyLink) PRD (제품 요구사항 정의서)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 개발자 및 크리에이터가 자신의 다양한 활동(GitHub, 포트폴리오, 유튜브, 블로그 등)을 하나의 통합된 웹페이지로 깔끔하게 모아서 공유할 수 있는 링크 모음 서비스.
- **대상 사용자**: 개발자, 크리에이터 등 실용적이고 직관적인 링크 모음이 필요한 개인.
- **기술 환경 정리**: Next.js, shadcn/ui 기반 단일 디자인 컴포넌트 적용, Firebase 생태계(구글 소셜 로그인, Firestore DB 연동)

## 2. 향후 확장 (Phase 2 계획)
- **방문자 통계**: 전체 페이지 조회수, 방문자 레퍼러(유입 경로), 및 각 링크별 클릭수 추적.
- **별도의 소셜 퀵 링크**: 메인 링크 리스트와 분리되어 깃허브, 트위터, 유튜브 등의 작은 로고 뱃지들을 별도로 배치할 수 있는 영역 제공.

## 3. 핵심 기능 목록

### 필수 기능 (MVP - Minimum Viable Product)
| 기능명 | 설명 | 중요도 |
| --- | --- | --- |
| **구글 소셜 로그인** | Firebase 연동 구글(Google) 소셜 로그인 단독 지원 구조 | High |
| **프로필 텍스트 관리** | 실제 표시될 '유저네임(이름)'과, 고유 URL용 '닉네임', '소개글' 설정 및 수정 기능 탑재 | High |
| **인라인 링크 관리** | 고유 URL과 타이틀을 가진 버튼 노출. 텍스트를 클릭해 현장에서 직접 수정(인라인 에디팅) 가능하도록 제공 | High |
| **닉네임 기반 퍼블릭 뷰** | 핸들(`@명칭`) 형식 없이, 설정한 고유 `닉네임` 자체가 내 페이지의 주소(`/닉네임`)로 동작 | High |
| **단일 디자인 (No Theme)**| 테마 설정 기능을 제거하고 shadcn/ui 기반의 고정된 단일 심플 레이아웃으로 통일 | High |
| **구글 API 파비콘 수집** | Google Favicon API를 직접 호출하여 링크 목적지의 아이콘을 자동으로 생성해 버튼 내에 배치 | High |
| **SEO 및 메타 태그 자동화** | 구글 프로필 이미지 및 유저네임을 이용해 외부 공유용 썸네일(OpenGraph 태그) 형태로 자동 전환 보장 | High |

*(참고: 기존에 논의되었던 '개발자 위젯 제공', '모바일 뷰 미리보기', '직접 이미지 업로드', '별도 테마 변경' 등의 기능은 제외되어 서비스가 훨씬 경량화되었습니다.)*

## 4. 기능 상세 설명

### 4.1 인증 및 URL 통제 시스템
- **단일 구글 로그인 시스템**: 회원가입/로그인은 Firebase 구글 계정 연동만 제공하여 과정을 단순화한다. 프로필 이미지 역시 구글 이미지를 가져다 렌더링한다.
- **역할이 분리된 닉네임과 유저네임 시스템**:
  - `닉네임(Nickname)`: **URL 주소 매핑 전용 텍스트**로 활용된다 (`mylink.com/닉네임`). 타인과 중복될 수 없으며, **최초 구글 연동 로그인 시 지메일(Gmail) 주소의 @ 앞부분(ID 영역)을 패치하여 기본값으로 자동 설정**한다. (만약 다른 유저가 이미 해당 텍스트를 선점한 상태라면 뒤에 난수를 붙여 생성하는 식의 처리 필요). 대시보드에서 언제든 수정은 가능하며 서버의 중복 체크를 거친다.
  - `유저네임(Username)`: URL과 관계없이, **프로필 상단에 예쁘게 표시되는 실제 사용자 이름**이다 (예: `홍길동`, 중복 가능).

### 4.2 유저 대시보드 (관리자 화면)
- **직관적 단일 뷰**: 모바일 뷰어 형태의 분리된 실시간 미리보기 기능 없이 전체 화면을 활용해 리스트 및 폼 위주로 쾌적하게 구성한다.
- **텍스트 프로필 에디터 (사진 업로드 불가)**:
  - 구글 프로필 이미지만 렌더링되며 별도의 로컬/서버 파일 업로드 기능을 제공하지 않는다.
  - **닉네임(URL용)**, **유저네임(실제 렌더링용 이름)**, **간단한 프로필 소개글(Bio)** 세 개 필드는 언제든지 자유롭게 수정할 수 있다.
- **인라인 (Inline) 링크 편집기**:
  - 저장된 링크 항목들의 리스트가 나열되며, 항목을 드래그 앤 드롭으로 순서를 바꿀 수 없다. (생성일 순 정렬)
  - 등록된 링크의 "제목(Title)"이나 "목적지 주소(URL)" 텍스트를 클릭하면 즉시 입력 폼 상태(인라인 편집 모드)로 전환된다.
  - 모달/팝업 이동 없이 현장에서 타건한 후, `Enter` 키나 영역 밖을 클릭 시 자동 저장 처리된다.

### 4.3 공개 프로필 (퍼블릭 뷰 페이지)
- **구글 Favicon API 연동**: 링크 버튼의 좌측 아이콘 영역은 `s2.googleusercontent.com/s2/favicons?domain=도메인주소&sz=64` 방식의 공개 API 주소를 `<img src="...">`에 통째로 바인딩하여 복잡한 로직 및 스토리지 처리 과정을 과감히 생략한다.
- **단일 테마 레이아웃 제공**: 테마 변경 제어 없이, 개발 시 확정된 가장 스탠다드한 shadcn/ui 기반 1가지의 중앙 정렬 카드형 디자인(모바일 친화)만 제공된다.
- **SEO (OpenGraph 태그 자동화)**: 퍼블릭 뷰 링크를 카카오톡, 슬랙 등에 공유할 때 노출되는 웹사이트의 메타 데이터 타이틀은 `<유저네임>의 마이링크`로 생성되고, 썸네일 이미지는 구글 계정에서 끌어온 아바타 URL로 자동 삽입되도록 SSR 세팅을 보장한다.
